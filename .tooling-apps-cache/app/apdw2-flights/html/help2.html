Flights Search

Submit a request to ./flights?code=<>&airline=<>&simulate=<>

Default Values are:

  • code: SFO
  • airline: american
  • mock: true

Allowed airport codes are CLE, LAX, PDX, and SFO. All other values default to SFO.

If simulate = true, then all three airline services are simulated locally within the Mule application. This allows the Flights search to work when the computer is offline. If mock = false, then external requests are made to the three MuleSoft training web services for simulated American, Delta, and United airlines flight information.

Here are some common links used in the course:

  • ./flights?code=SFO&airline=american
  • ./flights?code=LAX&airline=delta
  • ./flights?code=CLE&airline=united
  • ./flights?code=LAX&airline=all
  • ./flights?code=CLE&airline=united

Use real MuleSoft Training hosted web services

  • ./flights?code=SFO&airline=american&mock=false
  • ./flights?code=LAX&airline=delta&mock=false
  • ./flights?code=CLE&airline=united&mock=false
  • ./flights?code=LAX&airline=all&mock=false
  • ./flights?code=CLE&airline=united

Use solution flows with mocked endpoints

  • ./flights?solution=true&code=LAX&airline=delta
  • ./flights?solution=true&code=LAX&airline=all

Use solution flows with real MuleSoft Training hosted web services

  • ./flights?solution=true&code=LAX&airline=delta&mock=false
  • ./flights?solution=true&code=LAX&airline=all&mock=false
